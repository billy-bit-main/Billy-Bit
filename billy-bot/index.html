Billy Bot

basically alexa. except it's me talking to myself.

03

Setup

Full headless setup. No monitor. No keyboard. Just your computer and a terminal. If something breaks, it's almost always one of three things: wrong path, wrong device, or Bluetooth being Bluetooth.

01
Flash the OS

Download Raspberry Pi Imager. Select:

  • Device → your Pi
  • OS → Raspberry Pi OS Lite (64-bit)
  • Storage → your SD card

Open advanced settings (cog icon):

  • Set hostname → billybot
  • Enable SSH
  • Set username/password
  • Enter WiFi exactly

Click Write. Your SD card will disappear mid-process. That’s normal. Do not touch anything until it says complete.

02
Find the Pi

Wait 90 seconds. Then check your router devices list. Find billybot.

Alternative (sometimes works):

ping billybot.local

If that fails, ignore it. Router method is the only reliable one.

03
SSH in
ssh [username]@192.168.x.x

Use your IP. Hostnames sometimes fail. IP never does.

04
Update
sudo apt update && sudo apt upgrade -y
05
Dependencies
sudo apt install python3-pip espeak espeak-ng espeak-ng-data portaudio19-dev flac sox git -y

If audio or speech breaks later, it's almost always one of these missing.

06
Python packages
pip install SpeechRecognition pygame pyttsx3 requests vosk word2number --break-system-packages

That flag looks scary. It's fine. You need it.

08
Audio (Bluetooth)

This is the worst part of the entire build. If something randomly breaks later, it’s probably this.

bluetoothctl

Pair, trust, connect.

If it works once, good. It still might break later. That’s why you add a reconnect script.

09
Microphone
arecord -l

Look for your device. You’ll get something like: card 1, device 0

arecord -D hw:1,0 -f cd -t wav -d 3 test.wav

If this doesn’t work, your card number is wrong.

12
Transfer files

Run this on your computer. NOT inside SSH.

scp assistant.py user@ip:/home/user/billy/
13
Fix paths (critical)

If audio doesn’t play, this is why.

Every path must match exactly: /home/[username]/billy/Voices

If even one is wrong → silent failure.

15
Test
python3 assistant.py

If you see output but hear nothing → audio issue. If nothing runs → code issue.

16
Run on boot

If it doesn’t start on boot:

cat /home/[username]/billy/billy.log

Don’t guess. The log tells you exactly what broke.
